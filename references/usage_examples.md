# 多技能智能编排技能 - 使用示例

## 概述

本技能为aily工作助手提供了多技能智能编排能力，能够自动处理复杂业务需求，智能拆解任务步骤，匹配合适技能，编排最优执行链路。

## 核心功能

1. **智能任务拆解** - 将复杂需求分解为可执行的子任务
2. **技能匹配** - 根据子任务需求自动匹配合适的技能
3. **流程编排** - 生成最优执行顺序，支持并行执行
4. **异常处理** - 自动重试、错误恢复和流程纠正
5. **模板沉淀** - 将成功流程保存为可复用模板

## 使用场景示例

### 示例1：市场分析报告生成

**用户请求**：
```
帮我分析销售数据，找出销售额最高的产品类别和月度趋势，然后生成一份飞书云文档报告
```

**技能执行流程**：
1. 需求分析 → 检测到"数据分析"和"文档创建"需求
2. 任务拆解：
   - 数据清洗和预处理
   - 销售额分析
   - 趋势分析
   - 图表生成
   - 报告撰写
3. 技能匹配：
   - 数据清洗 → aily-data技能
   - 数据分析 → aily-data技能
   - 图表生成 → aily-chart技能
   - 报告撰写 → writer agent
4. 编排执行：
   ```
   数据清洗 → 数据分析 → 图表生成 → 报告撰写
   ```

2. 任务拆解：
   - 页面设计


## 最佳实践

### 1. 明确需求表达
- ✅ "帮我分析销售数据并生成可视化报告"
- ❌ "做一下销售分析"

### 2. 分阶段执行
- 复杂任务建议分阶段执行
- 可以先执行分析阶段，再执行生成阶段

### 3. 使用现有模板
- 检查是否有类似任务的现有模板
- 模板可以显著提高执行效率

### 4. 监控执行过程
- 关注错误日志和警告
- 及时调整执行策略

## 故障排除

### 常见问题

#### 问题1：技能匹配失败
**症状**: 无法找到合适技能执行子任务
**解决方案**:
- 检查技能描述是否准确
- 尝试手动指定技能
- 简化任务要求

#### 问题2：执行超时
**症状**: 任务执行时间过长
**解决方案**:
- 减少并行任务数
- 增加超时时间
- 优化任务拆分

#### 问题3：内存不足
**症状**: 内存错误或进程被杀死
**解决方案**:
- 减少数据处理量
- 使用流式处理
- 增加系统资源

## 扩展开发

### 添加新技能
1. 在`orchestrator_engine.py`中定义技能配置文件
2. 更新技能匹配逻辑
3. 添加相应的错误处理策略
